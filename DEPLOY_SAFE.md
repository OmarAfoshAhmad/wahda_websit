# 🛡️ دليل التحديث الآمن للإنتاج دون فقدان البيانات (Safe Production Deploy Guide)

لتحديث الموقع بالكامل (الواجهة الخلفية، الواجهة الأمامية، وتهجير قاعدة البيانات) بأمان تام ودون المخاطرة بحذف أي بيانات مستخدمين أو مستفيدين، يرجى الالتزام بالخطوات التالية.

---

## 🚫 السبب وراء حذف البيانات سابقاً
كان استخدام هذا الأمر هو السبب الرئيسي لحذف بيانات المستفيدين:
```bash
# ❌ أمر خطير جداً في الإنتاج ومخصص للتطوير فقط:
node node_modules/prisma/build/index.js db push
```
* **لماذا؟** لأن أمر `db push` يحاول مزامنة قاعدة البيانات مباشرة مع ملف `schema.prisma` دون تتبع تاريخ المايقريشنز. إذا وجد أي اختلاف أو تعارض في البيانات، فإنه يقوم بإسقاط الجداول (Drop Tables) أو تعديلها قسرياً مما يؤدي إلى مسح البيانات.

---

## 🟢 البديل الآمن والوحيد في الإنتاج:
```bash
#  الأمر الآمن لترحيل التحديثات دون مسح أي بيانات:
node node_modules/prisma/build/index.js migrate deploy
```
* **لماذا؟** لأن أمر `migrate deploy` يقرأ ملفات المايقريشنز المسجلة فقط ويطبق التعديلات الجديدة خطوة بخطوة مع الحفاظ الكامل على البيانات الموجودة.

---

## 📋 الخطوات الكاملة والآمنة للرفع والتحديث

### 💻 أولاً: على جهاز التطوير المحلي (Local)
عند إجراء أي تغيير في قاعدة البيانات (مثل إضافة عمود أو جدول جديد):
1. قم بإنشاء ملف ترحيل (Migration) جديد:
   ```bash
   npx prisma migrate dev --name add_new_features
   ```
2. تأكد من أن مجلد `prisma/migrations` قد تم إنشاؤه ويحتوي على ملف الـ SQL الجديد.
3. ارفع التعديلات كاملة إلى GitHub أو Gitlab:
   ```bash
   git add .
   git commit -m "تحديث الواجهات وقاعدة البيانات"
   git push origin main
   ```

---

### 🗄️ ثانياً: على سيرفر الإنتاج (Production VPS)
نفذ الأوامر التالية بالترتيب لتحديث الأكواد وتطبيق المايقريشن وإعادة تشغيل الموقع دون حذف أي بيانات:

#### 1. سحب التعديلات الجديدة:
```bash
git pull origin main
```

#### 2. بناء حاوية دكر الجديدة بالتعديلات (بدون كاش لضمان التحديث الكامل):
```bash
DOCKER_BUILDKIT=1 docker build --no-cache --secret id=build_env,src=.env.build -t wahda_web:latest .
```

#### 3. تشغيل أمر التهجير الآمن (Migration Deploy) لتعديل الجداول الحالية دون مسحها:
```bash
docker run --rm --env-file .env.production --network waadapp_tba_network wahda_web:latest node node_modules/prisma/build/index.js migrate deploy
```

#### 4. إعادة تشغيل حاويات المشروع لتشغيل الإصدار الجديد (Frontend & Backend):
```bash
docker compose -f docker-compose.prod.yml up -d --remove-orphans
```

---

## ⚡ فكرة ذكية: التحديث التلقائي عند الإقلاع (Migrations on Boot)
يحتوي ملف `Dockerfile` الخاص بنا بالفعل على ميزة لتشغيل التحديثات تلقائياً عند بدء الحاوية.
لتفعيلها، ما عليك سوى إضافة السطر التالي إلى ملف المتغيرات `.env.production` الموجود على السيرفر:
```env
RUN_DB_MIGRATIONS_ON_BOOT=true
```
* عند إضافة هذا الخيار، سيقوم التطبيق تلقائياً بتشغيل `migrate deploy` بأمان في كل مرة يتم فيها بناء أو إعادة تشغيل حاوية دكر، دون الحاجة لتشغيل أمر التهجير يدوياً!
